High impact

H1

Production build currently fails on a style budget

High · blocker ✓ Resolved

src/app/pages/article/article-detail/article-detail.page.ts:238-548 · angular.json:51-54

What
ng build errors: anyComponentStyle budget 4.00 kB not met by 1.20 kB (total 5.20 kB). The article page carries ~310 lines of inline component SCSS.
Why
No production APK can be produced right now. 5 kB of per-component CSS is also re-parsed on every style injection.
Fix
Move the comments-section CSS into _global.scss (where .gp-row/.gp-card already live) or split a comment-list child component. Don't just bump the budget — the file duplicates row/composer styles that exist globally.
Status
2026-07-06 — Hero, body, and comments-section CSS moved out of the component into _global.scss; component now carries only a one-line ion-content override. ng build --configuration production passes.
H2

Google Fonts fetched from the network in a native-only app

High ✓ Resolved

src/index.html:15-20 · src/theme/custom/base/_fonts.scss:7-8

What
A render-blocking <link> to fonts.googleapis.com pulls 3 families / 14 weights — IBM Plex Sans (4), Manrope (7, 200→800), JetBrains Mono (3). display=swap softens FOIT but not the network dependency.
Why
There is no hosted web version — every cold start on a low-end Android must reach Google's CDN before brand fonts render. First launch offline means system fallback fonts; flaky 3G stalls the WebView render path. Also a privacy / store-review smell.
Fix
Self-host woff2 subsets (Cyrillic + Latin — the app is Bulgarian-only) in assets/fonts/ with @font-face; trim Manrope to the weights actually used (400/600/700/800 — 200/300/500 are never referenced). Expect ~150–250 kB local total.
Status
2026-07-06 — Google Fonts <link>/preconnect tags removed from index.html. Each family now self-hosted as a single local variable-font file under src/assets/fonts/ (one @font-face per style, covering the full weight range) instead of the ~14 static per-weight/subset faces Google served. Note: shipped as variable TTFs rather than woff2 subsets, so the local payload is larger than the ~150–250 kB estimate — acceptable since it's bundled once into the app package, not downloaded per launch.

Medium impact

M1

Native keychain round-trip on every HTTP request

Medium ✓ Resolved

src/app/core/interceptors/auth.interceptor.ts:34 · secure-storage.service.ts:26-32

What
The interceptor awaits SecureStorage.get() (Keychain / EncryptedSharedPreferences over the Capacitor bridge) before every API call.
Why
Each request pays a JS→native→JS round-trip (several ms on low-end Android); forkJoin bursts like home (categories + articles) pay it twice concurrently.
Fix
Cache the token in memory in SecureStorageService — read the keychain once at bootstrap, update the cache in setToken/clearToken.
M2

Duplicate notifications request on first open

Medium

src/app/store/notifications/notifications.store.ts:126-139 · notifications.page.ts:111-115

What
The store self-loads when injected (TabsPage injects it for the unread badge), then the page's ngOnInit fetches page 1 again on every visit.
Why
Redundant network + JSON parse + state churn; pagination resets each tab visit.
Fix
Drop the page's ngOnInit fetch (keep pull-to-refresh), or add a stale-while-revalidate timestamp so loadNotifications skips fresh data.
M3

Onboarding PNGs are 3.5× oversized

Medium

src/assets/onb-1.png 163 kB · onb-2.png 109 kB · onb-3.png 154 kB · onboarding.page.ts:42,96-99

What
All three are 1125×950 RGBA rendered at max-width: 320px, with no width/height attributes on the <img>.
Why
~426 kB in the APK, and each decodes to a ~4 MB RGBA bitmap in WebView memory on a 360 px phone. Missing intrinsic dimensions shift layout while the first slide decodes.
Fix
Resize to ~640×540, convert to WebP (~25–40 kB each), add dimensions — or use NgOptimizedImage like gp-logo already does.
M4

Pull-to-refresh completes on a blind timer, not on data

Medium

home.page.ts:201-207 (1200 ms) · category-list.page.ts:180-186 · notifications.page.ts:127-132 · polls-list.page.ts:101-104

What
refresh() starts the store load, then completes the ion-refresher after a fixed setTimeout.
Why
On slow networks the spinner vanishes while content still loads (looks broken); on fast ones the user is held 1–1.2 s artificially. The same pages already do this correctly for infinite scroll (pendingInfiniteEvent + effect).
Fix
Reuse that pattern: stash the refresher event, complete it in an effect when loading() goes false.
M5

Unbounded DOM growth on infinite-scroll lists

Medium

category-list.page.ts:115-117 · search.page.ts:113-115 · notifications.page.ts:87-89

What
Rows accumulate forever at 12/page with no virtualization; each row renders a gp-blueprint SVG (~13 nodes) plus icons — 30+ DOM nodes per row.
Why
After 10–15 pages on a 2 GB Android device, hundreds of composited SVG textures degrade scroll FPS and memory. Fine with mock data; a real news archive will hit it.
Fix
CDK cdk-virtual-scroll-viewport for these three lists (Ionic's own guidance since ion-virtual-scroll was removed), or drop off-screen pages from the store.
M6

zone.js retained although the app is 100% signals + OnPush

Medium

src/main.ts:20-21 · src/polyfills.ts:50 · src/zone-flags.ts

What
Every component is OnPush, state is @ngrx/signals, there are no async-pipe patterns — yet zone.js (34.6 kB polyfill) still patches all timers and events, so every setTimeout and Capacitor callback schedules app-wide ticks.
Why
Wasted CPU per interaction on low-end devices, for change detection the signal graph already handles.
Fix
Trial provideZonelessChangeDetection() (Ionic ≥ 8.6/8.7 supports zoneless) and drop zone.js from angular.json:26-28. Test Ionic overlays + push listeners — their callbacks arrive outside Angular; the signal-based state makes this safe.

Function calls in template loops

Low

home.page.ts:95,118 · search.page.ts:94,217-220 · article-detail.page.ts:162,206,568 · add-news.page.ts:223

Low

What
Actual prod initial is 835.9 kB raw / 204.2 kB transfer; styles.css 54.4 kB; largest lazy chunk 19.3 kB. The 5 MB error would let the bundle sextuple silently.
Fix
Tighten to ~1 MB warn / 1.5 MB error. Ignore the 3 MB chunk in the stale www/ dev output — prod numbers are authoritative.

Low

What

L4

Splash / startup chain has an uncovered gap

Low

capacitor.config.ts (no SplashScreen plugin) · splash.page.ts:39-47 · auth.store.ts:169-205

What
Native splash auto-hides on defaults (~500 ms) → WebView boot → /splash route → AuthStore.bootstrap() (keychain + storage reads) → navigate. Possible white flash between native splash and web splash on slow devices. PreloadAllModules is fine here — files are local.
Fix
Add @capacitor/splash-screen with launchAutoHide: false; call SplashScreen.hide() from SplashPage once initialized() fires.
L5

Theme / metadata inconsistencies

Low

src/index.html:10 · global.scss:33-36 · tailwind.css:9-36 · src/assets/icon/

What
<meta name="color-scheme" content="light"> while variables.scss:92-117 implements full dark mode — UA surfaces (inputs, scrollbars) won't match. global.scss claims dark mode is "intentionally NOT used", contradicting the code and CLAUDE.md; no code ever sets the .dark class (no theme toggle exists yet). Tailwind's @theme duplicates the light palette by hand — dark values are not mirrored. Web-only icons (~128 kB) ship in the APK.
Fix
Set color-scheme: light dark, build the theme-toggle service CLAUDE.md requires, reconcile the comment, and exclude web manifest icons from the native copy.

Done well

  • ✓OnPush everywhere. Every real component declares it; state is exclusively @ngrx/signals + local signals — zero async pipes, zero manual markForCheck.
  • ✓Fully lazy routing. Every route is loadComponent (app.routes.ts:5-85), functional guards, withComponentInputBinding feeding signal inputs.
  • ✓Healthy code splitting. 204 kB initial transfer; each page a 1.5–5 kB lazy chunk.
  • ✓IONIC_IMPORTS does not defeat tree-shaking. A curated 9-component shell set; rare components stay page-local — and pages actually follow the rule.
  • ✓Near-zero icon bytes. No ionicons at runtime — one inline-SVG gp-icon path table plus gp-blueprint placeholders instead of raster images.
  • ✓List hygiene. Every @for has a stable track; skeletons on all loading states; aspect-ratio on all placeholders (no CLS).
  • ✓Textbook search. debounceTime(350) + distinctUntilChanged + switchMap + takeUntilDestroyed, min-2-chars gate, guarded pagination.
  • ✓Stores as cache. Home loads once and survives navigation; category pages merge; pagination guards prevent duplicate fetches; exhaustMap blocks double submits.
  • ✓Auth discipline. Single-flight serialized 401 refresh queue; token never sent to third-party hosts; auto-logout timer properly cleared.
  • ✓No leaks found. The one manual subscription is unsubscribed; push listeners registered once behind an idempotence flag; no setInterval anywhere.
  • ✓Zone hygiene meanwhile. zone-flags.ts disables custom-elements patching; eventCoalescing: true at bootstrap.
  • ✓gp-logo uses NgOptimizedImage with width/height/priority.
